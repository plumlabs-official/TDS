## 범례

- 🟢 = 있음
- 🟡 = 부분 (토큰만 / prop 구조 다름)
- ❌ = 없음

### 열 설명

- **Variants**: 컴포넌트 속성 (variant, size, state)이 shadcn 코드와 1:1 매핑되는지
- **Blocks**: 재사용 가능한 UI 블록 (Hero, Pricing, Dashboard, 모바일 등)
- **컬러**: Variable collection으로 관리되는 색상 시스템
- **타이포**: Variable collection 또는 Text styles로 관리되는 폰트 시스템
- **간격**: spacing 토큰 (gap, padding, margin 등)
- **아이콘**: 아이콘 라이브러리 포함 여부
- **다크모드**: Variable mode로 Light/Dark 전환 지원
- **Auto Layout**: 컴포넌트에 Hug/Fill, Gap, Padding 적용 여부

## 유료 Kit 상세 비교

### shadcn 오리지널과의 호환성

| 항목 | shadcndesign | Shadcraft | Shadcn Studio |
|------|--------------|-----------|---------------|
| **Variant prop명** | Variant ✅ | Type ⚠️ | style ⚠️ |
| **Size** | Size ✅ | Size ✅ | size ✅ |
| **State** | State ✅ | State ✅ | state ✅ |
| **추가 prop** | Kbd | Kbd, Icon-only, Lucide | shape, Badge, info/warning/success |
| **토큰 네이밍** | spacing/, height/, border-radius/ | height/, radius/, p-# | spacing/, border radius/ |

**shadcn 오리지널 코드:**
```tsx
<Button variant="destructive" size="lg">
```

**결론:** shadcndesign이 코드 1:1 매핑에 가장 충실함.
