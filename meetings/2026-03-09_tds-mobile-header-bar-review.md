## Phase 1: 전문가별 분석

### [DESIGN DIRECTOR]

**긍정:**
- 커스텀 Frame → 디자인 시스템 인스턴스 전환으로 일관성 보장
- 4 Variant 구조가 iOS HIG Navigation Bar 패턴과 유사
- Status/Streaks 배지 별도 컴포넌트 분리 → 재사용성

**우려:**
1. 아이콘 소스 혼용 (Lucide + Huge Icons) — **높음**
2. Subtitle 프로퍼티 visible이나 SESSION.md에 삭제 TODO — 중간
3. Title 20px → 18px 차이 — 낮음
4. Button 터치 영역 36px < AA 기준 44px — **높음** (Button 컴포넌트 레벨)

### [ENGINEERING LEAD]

**긍정:**
- Variant → React type prop 1:1 매핑 가능
- Boolean 프로퍼티 → 조건부 렌더링 직관적
- Ghost/Icon Button 재사용으로 의존성 명확

**우려:**
1. Right 영역 Button 3개 하드코딩 → Boolean 토글 필요
2. Status/Streaks 내부 래스터 이미지 → SVG/아이콘 권장
3. Subtitle visible 상태에서 실제 사용 여부 미결정

### [PRODUCT LEADER]

**긍정:**
- Screen+Actions variant = 탭 기반 메인 화면에 적합
- Streaks 배지 상시 가시화 = 동기부여 요소

**우려:**
- 60px 안에 Title + Subtitle + Streaks + Actions 3개 = 과밀
- 검색바 통합 헤더는 미커버 (별도 컴포넌트로 대응 가능)

## Phase 2: 종합 토론

### 합의점
- 커스텀 Frame → TDS 컴포넌트 인스턴스 전환은 올바른 방향
- 4 Variant 구조는 모바일 앱 헤더 사용 케이스를 잘 커버

### 충돌점

- Right Button 3개 고정: 전원 Boolean 토글 필요 동의
