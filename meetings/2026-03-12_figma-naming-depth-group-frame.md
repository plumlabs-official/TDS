## Q1. 인스턴스 내부 노드까지 네이밍 커스텀해야 하나?

**답: 안 함. 컴포넌트 메인에서 관리.**

| 레벨 | 커스텀 | 이유 |
|------|--------|------|
| Screen 프레임 | 필수 | 화면 식별 |
| 섹션 컨테이너 | 필수 | 구조 파악 |
| 인스턴스 자체 | 선택 | 가독성 |
| 인스턴스 내부 자식 | 안 함 | 컴포넌트 관리 영역 |

- 인스턴스 내부 rename → 컴포넌트 업데이트 시 리셋 가능
- Figma AI rename도 인스턴스 내부 미처리

## Q2. 인스턴스 이름 바꿔도 되나?

**답: 안전함 (ID 기반 링크).**

- 컴포넌트 링크 유지됨
- "Reset name"으로 원복 가능
- 단, 메인 컴포넌트 rename과 혼동 주의 (메인 rename은 라이브러리 링크 끊길 수 있음)

## Q3. Group vs Frame 기준

**답: Frame이 기본. Group은 순수 시각 묶음일 때만.**

| Group | Frame |
|-------|-------|
| 벡터/도형 묶기 | UI 레이아웃 |
| 자식에 맞춰 자동 크기 | 독립적 크기 |
| Auto Layout 불가 | Auto Layout 가능 |
| Fill/Stroke 불가 | 모든 스타일 가능 |
