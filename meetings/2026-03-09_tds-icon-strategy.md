## 배경

Shadcn Kit의 아이콘은 Huge Icons 라이브러리의 인스턴스 상태. TDS에 복사하면 외부 의존성 문제 발생. Tryve 프론트엔드(React + shadcn/ui) 개발 효율성을 고려한 결정 필요.

## 옵션 분석

### A: Detach → 벡터 독립
- TDS 완전 독립, 외부 의존 0
- 수백 개 아이콘 수동 관리, 코드 매칭 없음
- 유지보수 비용 높음

### B: Lucide Icons 통일 (추천)
- shadcn/ui 공식 아이콘 셋, lucide-react 1:1 매칭
- Tree-shaking, TypeScript 자동완성
- currentColor로 TDS 변수 자연 연동
- 없는 아이콘은 Lucide 규격(24x24, stroke 2px)으로 커스텀 제작

### C: Huge Icons 라이브러리 연결 유지
- 최대 아이콘 수, 자동 업데이트
- shadcn 생태계 불일치, 별도 패키지 필요

## 결론

**B (Lucide Icons)로 전원 합의.**

핵심 근거: Figma ↔ 코드 1:1 매칭이 디자인 시스템의 최우선 기준. shadcn/ui가 Lucide를 공식 채택했으므로 생태계 정합성 확보.
