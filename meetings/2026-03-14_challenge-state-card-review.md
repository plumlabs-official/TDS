## Phase 1: 전문가별 분석

### Design Director

**Atomic Design 프레임워크 적용:**

Content Header = Molecule 레벨 (Atom 조합: Title, Description, Button)

| 항목 | 기준 | 현황 | 판정 |
|------|------|------|------|
| 재사용 횟수 | 3회 이상 → 컴포넌트화 | 다수 사용처 | PASS |
| 토큰 사용 | 디자인 토큰만 사용 | 토큰 기반 | PASS |
| 간격 규칙 | 4px/8px 그리드 | gap 6/12px, padding 12px | PASS |
| 아이콘 세트 | 하나의 아이콘 세트 | 4종 혼용 | **FAIL** |

Content Header 정당성:
1. 레이아웃 캡슐화 — Left(row) vs Center(column) 전환
2. Size별 타이포 일관성 — Large=2xl, Medium=lg, Small=base
3. Slot 패턴 — Avatar Group 등 자유 주입

문제는 Card 레벨: 아이콘 4종 혼용, Button 네이밍 중복, Card Header 미컴포넌트화

### Engineering Lead

**Build vs Buy 평가:**
- 고유 요구사항 (Alignment 전환 + Slot): Build +2
- shadcn 동등 컴포넌트 없음: Build +2
- 3회 이상 재사용: Build +1
- Build 합계: 5 → 커스텀 컴포넌트로 Build

shadcn CardHeader는 스타일링 래퍼(`flex flex-col space-y-1.5 p-6`)이지 레이아웃 컴포지션이 아님. Content Header와 레벨이 다름.

기술 부채:
| 부채 유형 | 영향도 | 우선순위 |
|----------|--------|----------|
| 아이콘 4종 혼용 | 높음 | 즉시 해결 |
| Card Header 미컴포넌트화 | 낮음 | 백로그 |
| Button 네이밍 중복 | 중간 | 다음 스프린트 |
| Root mode:none | 낮음 | 허용 |
