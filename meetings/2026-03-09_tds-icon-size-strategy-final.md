# TDS 아이콘 사이즈 전략 최종 결정

- **날짜**: 2026-03-09
- **모드**: Team Mode — Trade-off Discussion
- **참석**: Design Director, Engineering Lead
- **주제**: 아이콘 사이즈 관리 — 라이브러리 원본 변경 vs Button 컴포넌트 제어 vs variant 추가

---

## 배경

- TDS 아이콘 라이브러리: 24×24 (Lucide + Huge Icons)
- Button `Size=Icon` 내부 아이콘이 16×16으로 표시되어 너무 작음
- 사용자가 Button 원본을 수정: padding 8→4, 아이콘 fill, 36×36

## 검토한 옵션

| 옵션 | 설명 | 결과 |
|------|------|------|
| 라이브러리 원본 32로 변경 | 모든 아이콘 컴포넌트 리사이즈 | **기각** — 사용처마다 사이즈 다름 |
| Button 원본 수정 (채택) | Size=Icon을 36×36, padding 4로 | **채택** — 주 사용처(앱바)에 맞춤 |
| Size=Icon-sm variant 추가 | 작은 사이즈용 별도 variant | **기각** — YAGNI, 필요 시 오버라이드 |

## 최종 결정

**패턴 B: 원본 = 주요 사용처, 예외는 인스턴스 오버라이드**

- 아이콘 라이브러리: 24×24 유지
- Button `Size=Icon`: 36×36, padding 4px, 아이콘 fill (28×28 실질)
- 작은 사이즈 필요 시: 사용처에서 인스턴스 오버라이드
- 3곳 이상 반복되면 variant 재검토

## 코드 매핑

```tsx
// size="icon" → 36×36, padding 4px
const buttonVariants = cva("...", {
  variants: {
    size: {
      icon: "h-9 w-9 p-1",
    }
  }
});

// 아이콘은 부모 fill
<Button size="icon" variant="ghost">
  <Menu className="w-full h-full" />
</Button>
```
