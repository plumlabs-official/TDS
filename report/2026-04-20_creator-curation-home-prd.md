## TL;DR

- **목적**: 크리에이터 자체(사람)에 집중한 큐레이션으로 팔로우·발견 동력 확보
- **타겟**: 참여/팔로우 이력 없는 초기 유저, 크리에이터 탐색 의도 유저
- **결과물**: 모바일 단일 스크롤 화면 1개 (라운지 홈과 **별개 페이지**)
- **핵심 제약**: **(1) 별도 랜딩 페이지 없음 — 모든 카드 탭 시 해당 크리에이터 라운지 직행 (운영 코스트 최소화)**, **(2) 챌린지 정보 카드 노출 금지 — 크리에이터 자체에 집중**
- **Claude Design 테스트 목표**: CDS 라이브러리 인식도 평가 (인스턴스 활용율, 디자인 토큰 바인딩, 네이밍 시맨틱, IBM Plex Sans KR 폰트 렌더링)

## 0. 핵심 제약 원칙 [확정]

### 0.1 별도 랜딩 페이지 없음 — 카드 탭 시 크리에이터 라운지 직행

**원칙**: 어떤 섹션의 어떤 카드든 탭하면 **곧바로 해당 크리에이터의 라운지**로 이동한다. 중간에 스토리텔링 페이지/큐레이션 에세이/피처드 상세 같은 별도 랜딩 페이지를 만들지 않는다.

- **이유**: 운영 코스트 최소화. 에디터가 별도 페이지 콘텐츠를 만들어야 하는 스펙 없음.
- **허용**: 에디터 픽(수동 피처드) 자체는 허용. 단, 에디터가 하는 작업은 "어떤 크리에이터를 이 섹션에 노출할지 선정" 수준까지만.
- **금지**: 앱스토어 Today식 "Featured Story 페이지" (편집팀이 별도로 꾸민 전체 화면 상세), 피처드 스토리 랜딩, 큐레이션 에세이 페이지
- **예외**: 카테고리 Chip 필터 적용/해제는 동일 페이지 내 스크롤/필터링 (별도 페이지 아님) → 허용
- **UI 영향**: `더보기` 링크가 있다면 전체 리스트 뷰(단순 리스트)까지만 허용. 스토리/에세이 페이지로 연결 금지.

### 0.2 크리에이터 중심 — 챌린지 정보 노출 금지

**원칙**: 카드에 **챌린지 정보(제목 / 참여자 수 / 기간 / 챌린지 개수)를 노출하지 않는다.** 크리에이터 그 자체(프로필, 페르소나, 팔로워)에 집중.

- **이유**: 챌린지 홈과 중복/차별화 부재 방지. 이 화면은 "사람 발견" 지면.
- **카드 구성 (공통)**:
  - 프로필 비주얼 (아바타 또는 프로필 이미지)
  - 이름 + 티어 배지 (Pro / Lite)
  - AI 생성 한 줄 소개 (페르소나 중심)
  - 카테고리 태그 (1~2개)
  - 팔로워 수
  - 팔로우 버튼
- **금지 필드**: 챌린지 제목, 챌린지 썸네일, 챌린지 참여자 수, 챌린지 개수, 챌린지 완주율
- **예외 고려**: 활동성 시그널이 필요하면 "최근 소식 N개" 같은 라운지 활동 메타로 대체 가능 (챌린지 아님)

## 1. 배경 및 문제 정의

### 1.1 문제

> 미팅 §9 인용: "초기 유저가 참여/구독 없이 개별 크리에이터 글을 본들 의미가 크지 않음. 크리에이터 영입 동력도 필요."

- 라운지 홈은 **내가 팔로우한 크리에이터 중심** (내 영역)
- 초기 유저는 "볼 게 없는" 상태 → 이탈 리스크
- 신규 크리에이터 입장에서도 "핫하게 보이는" 발견 지면 부재 → 영입 설득력 약함
- 챌린지 홈은 이미 챌린지 중심 발견 지면 → 또 다른 챌린지 피처드는 불필요

### 1.2 해결 컨셉

앱스토어 Today 탭 포맷에서 영감을 받되, **크리에이터(사람) 중심**으로 재해석한 발견 지면. 에디터 픽 + 자동 랭킹 혼합 큐레이션으로 구성하되, **카드 → 라운지 직행** 구조로 별도 랜딩 페이지 제작 코스트 제거.

### 1.3 비즈니스 시나리오

- 피처드 실적 스토리텔링 → 인플루언서 섭외 시 협상 카드
- 미니멈 개런티 + 본인 채널 홍보 바터
- 별도 상세 페이지 제작 코스트 없이 생태계 선순환

## 2. 타겟 사용자 및 진입 맥락

| 페르소나 | 니즈 | 진입 기대 |
|---------|------|----------|
| **초기 유저 A** (참여/팔로우 0) | 누구를 팔로우할지 모름 | 피처드 크리에이터 발견 → 카드 탭 → 라운지 진입 → 팔로우 |
| **초기 유저 B** (무료 챌린지만 이용) | 관심 주제별 크리에이터 탐색 | 카테고리 그루핑 브라우징 → 라운지 진입 |
| **활성 유저** (다른 크리에이터 탐색 의도) | 새 분야 도전 | 오늘의/금주의 크리에이터 확인 → 라운지 진입 |

## 3. 정보 구조 (IA)

```
크리에이터 큐레이션 홈 (단일 스크롤)
├── Top Bar (타이틀 "발견" 또는 "크리에이터")
├── [검토중] 카테고리 Chip Bar (가로 스크롤 필터 — 같은 페이지 내 필터링만)
├── Section 1. 오늘의 크리에이터 (Hero, 1명 — 에디터 픽 또는 자동)
├── Section 2. 금주의 크리에이터 (가로 스크롤, 5~7명 — 자동 랭킹)
├── Section 3. 주제별 큐레이션 (다이어트 / 명상 / 아침 도전 / 러닝 / 습관 리셋 등)
│   └── 섹션별: 헤더 + 가로 스크롤 카드 5~7명
└── [검토중] Section 4. 급상승 크리에이터 (자동 랭킹)
```

**모든 카드 탭 액션 = 해당 크리에이터 라운지로 직행** (제약 0.1)
**[제거됨]**: 피처드 스토리 상세 페이지, 큐레이션 에세이 페이지

## 4. 섹션 상세

### 4.1 Hero — 오늘의 크리에이터

- **카드 형태**: 풀블리드 프로필 비주얼 (16:9 또는 4:5)
- **구성 요소**:
  - 상단 태그 Chip: "오늘의 크리에이터"
  - 크리에이터 프로필 (프로필 이미지 + 이름 + 티어 배지 Pro/Lite)
  - AI 생성 한 줄 캐치 카피 (예: "아침 루틴으로 삶을 바꾸는 웰빙 코치")
  - 카테고리 태그 1~2개
  - 팔로워 수
  - CTA 버튼: `팔로우` (secondary variant)
- **선정 기준**: 에디터 픽 또는 자동 스코어링 (24h 활동성 + 팔로워 증가율)
- **상태**: 팔로우 전 / 후 (토글)
- **탭 액션**: **카드 탭 → 해당 크리에이터 라운지로 직행** (중간 상세 페이지 없음)

### 4.2 금주의 크리에이터

- **레이아웃**: 가로 스크롤 캐러셀, 카드 5~7개
- **카드 구성** (중형):
  - 프로필 비주얼 (1:1 또는 4:5)
  - 이름 + 티어 배지
  - AI 생성 한 줄 소개 (페르소나 중심)
  - 카테고리 태그 1~2개
  - 팔로워 수
  - `팔로우` 버튼
- **섹션 헤더**: "금주의 크리에이터" + 서브카피 + `더보기` 링크 (→ 전체 리스트 뷰만, 스토리 페이지 금지)
- **선정 기준**: 7d 통합 랭킹 점수 자동 (팔로워 증가 + 소식 반응)
- **탭 액션**: 카드 탭 → 해당 크리에이터 라운지 직행

### 4.3 주제별 큐레이션

- **카테고리 예시**: 다이어트 / 명상 / 아침 도전 / 러닝 / 습관 리셋
- **섹션 구조** (반복):
  - 섹션 헤더: 카테고리명 (아이콘 + 타이틀) + 서브카피 + `더보기`
  - 가로 스크롤 카드 5~7개 (중형, 4.2와 동일 구조)
- **선정 기준**: 크리에이터 본인 카테고리 태그 + 해당 카테고리 내 랭킹 점수 자동
- **탭 액션**: 카드 탭 → 해당 크리에이터 라운지 직행

### 4.4 [검토중] 급상승 크리에이터

- 최근 7일 팔로워/소식 반응 증가율 자동 랭킹
- 소형 카드 가로 스크롤 (프로필 + 이름 + 배지 + 증가율 인디케이터)
- **주의**: 증가율 인디케이터에 챌린지 관련 지표(참여자 증가) 사용 금지 → 팔로워/소식 반응 지표만
- **탭 액션**: 카드 탭 → 해당 크리에이터 라운지 직행

## 5. CDS 컴포넌트 매핑

> **원칙**: 기존 CDS 인스턴스 최대 활용. 신규 컴포넌트는 Hero/Creator Card 2종에 한정.

| UI 요소 | CDS 컴포넌트 | 확장/신규 여부 | 메모 |
|--------|-------------|---------------|------|
| 섹션 헤더 | `Content Header` | 기존 | 타이틀 + 서브타이틀 + 더보기 링크 |
| Hero 카드 | **`Creator Hero Card`** (신규 제안) | 신규 | Profile Card 구조 확장 + 풀블리드 프로필 비주얼 |
| 금주/주제별 카드 | **`Creator Discover Card`** (신규 제안) | 신규 | Profile Card(Vertical) 파생. 프로필 비주얼 + 이름 + 소개 + 팔로우 |
| 티어 배지 | `Creator Tier Badge` (D11 예정) | 신규 | Pro/Lite 별개 심볼 체계 (2026-04-15 미팅 결정) |
| 팔로우 버튼 | `Button` (variant=secondary, size=sm) | 기존 | 토글 상태 |
| 카테고리 Chip / 태그 | `Badge` or `Chip` | 기존 | 카드 내부 태그 + 상단 필터 공용 |
| 메타 텍스트 (팔로워 수) | `Meta Text` 패턴 | 기존 | 아이콘 + 수치 |
| 섹션 구분 | Auto Layout + spacing 토큰 | N/A | gap: 24~32 |

### 5.1 신규 컴포넌트 정의 후보

#### `Creator Discover Card` (중형, 가로 캐러셀 반복)

- Root: VERTICAL Auto Layout, padding 0, gap 8
- Profile Visual Slot (1:1 또는 4:5, radius `--radius/md`)
- Content (VERTICAL, gap 4)
  - Name Row (HORIZONTAL, gap 6): Name + Tier Badge
  - Description (AI 생성 한 줄 소개, 최대 2줄 ellipsis)
  - Meta Row (HORIZONTAL): Category Tag(1~2) + Follower Count
- Footer: Follow Button
- **Boolean props**: `Show Category Tag`, `Show Follower Count`, `Show Follow Button`
- **Variant props**: `Size` (sm/md/lg), `Is Followed` (true/false)
- **금지 슬롯**: Challenge Preview 슬롯 없음 (제약 0.2)

#### `Creator Hero Card` (대형, Hero 섹션 1회)

- Root: VERTICAL Auto Layout, padding 0
- Full-bleed Profile Visual (16:9 또는 4:5, scrim 그라데이션)
- Overlay Content (absolute 허용 — 풀블리드 비주얼 위)
  - Top Chip: "오늘의 크리에이터"
  - Name + Tier Badge
  - AI 생성 한 줄 캐치 카피
  - Category Tag + Follower Count
- Footer Action: Follow Button

## 6. 인터랙션 및 상태

## 7. 비주얼 톤 가이드

> Challify 웰니스 톤(밝고 경쾌)을 유지. 페이월 리디자인(D8)과 동일 톤 기조.

- **배경**: `--background` (화이트 베이스)
- **Hero 카드**: 크리에이터 테마 컬러 액센트(프로필 기반 자동 추출, 시맨틱 토큰 내)
- **타이포**: IBM Plex Sans KR (Pretendard 복원 시 교체)
- **이미지 처리**: cornerSmoothing 0.6, radius `--radius/md` (12px) 기본
- **그라데이션**: Hero 프로필 비주얼 하단 scrim (텍스트 가독성 확보)
- **섀도우**: 카드 `shadow/sm`, Hero만 `shadow/md`
- **토큰 바인딩 필수**: color, spacing, radius 모두 CDS variable 바인딩 (하드코딩 금지)

## 8. AI 생성 카피 가이드라인

### 8.1 크리에이터 한 줄 소개 (Hero / Discover Card 공통)

- 형식: "{동사} {대상}의 {페르소나/전문성}"
- 예시:
  - "아침 루틴으로 삶을 바꾸는 웰빙 코치"
  - "지속 가능한 다이어트를 설계하는 영양 전문가"
  - "10분 명상으로 하루를 리셋하는 마인드풀 가이드"
- 제약: 40자 이내 권장, 최대 2줄
- **금지**: 챌린지 제목/스펙 인용 금지 (제약 0.2)

### 8.2 섹션 서브카피

- 오늘의 크리에이터: "편집팀이 주목하는 크리에이터" 또는 "지금 가장 주목받고 있어요"
- 금주의 크리에이터: "이번 주 활발하게 활동 중"
- 주제별: "{주제}에 진심인 크리에이터"

## 9. 배치 위치 옵션 [보류]

미팅 §9 보류 항목. 시안 확정 후 재판단.

| 옵션 | 장점 | 단점 |
|------|------|------|
| A. 피드 탭 내 상단 섹션 | 기존 탭 유지, 빠른 접근 | 피드 퍼스트 인상 약화 |
| B. 별도 하단 탭(‘발견’) | 독립 지면, 탐색 몰입 | 탭 개수 증가 |
| C. 홈 진입 시 첫 화면 | 초기 유저 강제 노출 | 기존 유저 스크롤 피로 |

**현재 기본**: A (피드 탭 내 상단 섹션) — 시안 검증 후 B 전환 가능성 열어둠

## 11. Claude Design 테스트 요청 프롬프트

> Claude Design에 CDS 인식도 평가를 위해 입력할 프롬프트 초안.

```
Challify의 "크리에이터 큐레이션 홈"을 디자인해줘.

컨셉: 앱스토어 Today 스타일이되, 크리에이터(사람) 중심 발견 지면. 웰니스/건강 도메인.
타겟: 팔로우 이력 없는 초기 유저.

핵심 제약:
1. 별도 랜딩 페이지 없음. 모든 카드는 탭 시 해당 크리에이터 라운지로 직행. 피처드 스토리/큐레이션 에세이 같은 상세 페이지 만들지 마.
2. 카드에 챌린지 정보(제목/참여자 수/기간) 노출 금지. 크리에이터 자체에 집중.

화면 구성 (위에서 아래 단일 스크롤):
1. Top Bar — 타이틀 "발견"
2. Hero — 오늘의 크리에이터 (풀블리드 프로필 비주얼 16:9, 이름 + Pro 배지 + AI 생성 한 줄 소개 + 카테고리 태그 + 팔로워 수 + 팔로우 버튼. 하단 scrim 그라데이션)
3. 금주의 크리에이터 — 섹션 헤더 + 가로 스크롤 카드 5개 (프로필 1:1, 이름 + 배지 + 한 줄 소개 + 카테고리 태그 + 팔로워 수 + 팔로우 버튼)
4. 주제별 큐레이션 (반복) — 다이어트, 명상, 아침 도전 (섹션 헤더 + 가로 스크롤 카드 5개)

카드 필드는 오직 크리에이터 정보만: 프로필 이미지, 이름, 티어 배지(Pro/Lite), AI 생성 한 줄 소개, 카테고리 태그 1~2개, 팔로워 수, 팔로우 버튼.
챌린지 썸네일/제목/참여자 수/완주율은 어디에도 표시하지 마.

CDS 라이브러리의 Profile Card, Button, Badge, Content Header를 최대한 활용.
폰트: IBM Plex Sans KR. 색상/간격/라디우스는 CDS variable로 바인딩.
톤: 밝고 경쾌한 웰니스. 카드는 soft shadow + radius 12.

가상 크리에이터는 다이어트/명상/러닝/습관 카테고리별 3~5명씩 생성.
```

## 12. PoC 평가 체크리스트

Claude Design 결과물을 받으면 아래 축으로 점검:

**CDS 인식도**
- [ ] CDS 인스턴스 활용: Button/Profile Card/Badge/Content Header를 실제 인스턴스로 배치(detach 없음)
- [ ] 토큰 바인딩: color/spacing/radius가 CDS variable에 바인딩 (하드코딩 hex 없음)
- [ ] 폰트: IBM Plex Sans KR 렌더링 확인
- [ ] 네이밍 시맨틱: 레이어명이 `Hero`, `Section`, `Card` 등 시맨틱 어휘 (금지 접미사 없음)
- [ ] Auto Layout: flex/grid 1:1 매핑 가능 (불필요 absolute 없음, Hero scrim overlay 제외)

**제약 준수**
- [ ] 별도 랜딩 페이지 생성 안 됨 (모든 카드 탭 액션이 크리에이터 라운지 직행만)
- [ ] 카드에 챌린지 정보 없음 (챌린지 제목/썸네일/참여자 수/개수/완주율)
- [ ] 카드 필드가 크리에이터 중심 (프로필 + 이름 + 배지 + 소개 + 태그 + 팔로워 + 팔로우)

**디자인 품질**
- [ ] Creator Discover Card / Creator Hero Card 구조가 CDS 제출 고려 가능한 품질인가
- [ ] 핸드오프 번들(Claude Code 연계 시) React+Tailwind 코드 품질 — CDS 토큰 매핑 정확도
